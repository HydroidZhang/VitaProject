## 2. 实现思路

### 2.1 技术选型

- **Godot 4.6**：2D 竖屏手游上手快，场景化 UI 适合还原参考图三屏结构。
- **分层架构**：数据层（牌型、关卡）→ 生成层（布局、洗牌、可解性）→ 表现层（牌面、HUD、音效），便于分阶段用 AI 辅助实现和排查。
- **OpenSpec**：用 spec-driven 方式记录 P0～P5 各阶段需求与归档，避免「只有代码、没有设计依据」。

### 2.2 AI 工具使用方式

本次开发 **大量使用 Cursor Agent** 辅助，但关键决策与验收仍由人工完成：

| 环节 | AI 辅助 | 人工负责 |
|------|---------|----------|
| 架构与模块拆分 | 生成方案、补全样板代码 | 确认分层边界、Godot 节点结构 |
| 玩法规则 / 可解性 | 实现 `BoardRules`、`BoardSolver` 草稿 | 真机试玩、边界 case |
| UI 场景 | 生成 `.tscn` 骨架与脚本绑定 | **在编辑器里拖布局**、对参考图微调 |
| 资源 | AI 批量生图、自动切图脚本 | **PS 逐张修图**、统一尺寸与命名 |
| 文档 | README、OpenSpec、本总结 | 删减、对齐题目与真实实现 |

原则：**玩法与输入链路必须可玩**；美术先程序化生图/白模保底，再换 PS 修过的 PNG。

### 2.3 核心流程

```
启动 Main.tscn
  → 主页（继续当前解锁关 / 选关 / 设置）
  → 对局（程序化生成关卡 → 点选配对消除）
  → 通关结算（时间 / 分数 / 连击 → 下一关或回主页）
```

20 关不依赖 20 份手写布局 JSON，而是：

- **固定 5×7 槽位**（`GridSlots`）：底层 35 格 + 半格偏移中层 + 顶层；
- **按关卡递增层数与牌数**（`LayoutGenerator` + `LevelCatalog`）；
- **洗牌后做可解性验证**（`BoardSolver`），失败则重试。

进度保存在 `user://vita_progress.json`，通关解锁下一关。

## 3. 功能清单（对照参考图）

### 首页

- 木纹背景、Logo、门环装饰、橙色「关卡 N」大按钮、选关入口、设置按钮。
- 启动播放主界面 BGM（`BgmManager.play_home()`）。

### 游戏页

- 顶栏：返回、关卡 / 分数 / 匹配数、菜单（打开选关）。
- 牌桌：多层叠牌、贴图或程序化牌面、金黄选中框 + 绿配对提示、撞合碎裂消除动画。
- 底栏：洗牌、提示（带剩余次数角标）；**洗牌时牌先收拢到中心，再重新展开发牌**。
- 点击被挡牌时顶栏下飘字（「该麻将被盖住」「两边被锁住」）。
- 配对成功在碰撞点上方飘分；**3 秒内连续消除**触发右侧「N连击!」提示与连击加分。

### 结果页

- 「才华横溢」标题、时间 / 分数 / **最高连击**统计、分级鼓励文案、绿色进度条与下一关按钮、返回主页。
- 通关后解锁下一关并持久化。

### 设置

- 大号音乐开关（绿/灰切换）、橙色音量进度条 + 叠层滑块，适配手指操作。

### 选关

- 20 关列表，显示名称、难度星级、锁定状态。
- 已解锁进度提示「已解锁 X / 20 关」。

### 界面切换

- 主页 / 游戏 / 选关 / 结算 / 设置之间有过渡动画，避免硬切。
- 主页相关切换采用 **交叉淡入淡出**（`ScreenTransition.crossfade`），不用全屏缩放，观感更柔和。
- 结算与设置弹窗：遮罩淡入 + 内容区中心展开。

## 4. 创新与体验优化（题目鼓励项）

1. **ScrollListTap（选关列表）**  
   对标 Android `RecyclerView` touch slop：手势在 `ScrollContainer` 统一处理，关卡行 `mouse_filter=IGNORE`，**滑动与点击分离**，解决「在按钮上滑不动」和「滑动误触进关」。已写入 `openspec/specs/level-progression/spec.md`。

2. **程序化 20 关**  
   槽位 + 层数分布 + 可解性验证，避免手工维护 20 个布局文件，难度随关卡 ID 递增。

3. **场景驱动 UI**  
   布局、贴图、StyleBox 均在 `.tscn` 中编辑；脚本只更新文案与数值，避免运行时拼 UI 导致调位置无效。

4. **音频与设置**  
   主页 / 进关 / 通关三套 BGM；音乐开关与音量写入 `user://vita_settings.json`。

5. **热更新预留**  
   `HotUpdateManager` + PCK 热更管线（作业场景可不启用远程 CDN）。

6. **连击系统**  
   3 秒窗口内连续消除计连击；右侧单次飘字提示（不与分数飘字重复）；连击加分计入总分；结算展示最高连击。见 `openspec/specs/tile-gameplay/spec.md`、`game-ui/spec.md`。

7. **选中与碰撞手感**  
   选中：加粗金黄边框、牌面黄蒙层、粒子闪烁、金色侧棱；可配对：薄荷绿蒙层 + 细绿框。消除：两牌撞合后压扁弹开，碎屑与粒子挂在 `TileLayer` 坐标，牌桌微震。见 `openspec/specs/tile-visuals/spec.md`。

8. **设置页大控件**  
   音乐大按钮、橙色音量条，解决移动端点不中细滑条的问题。见 `openspec/specs/game-ui/spec.md`。

9. **洗牌动画**  
   点击洗牌后，当前牌面先向牌桌中心收拢叠起，后台重新生成可解布局，再从中心依次展开到新位置；洗牌期间屏蔽点牌与提示。见 `Scripts/Visual/shuffle_effect.gd`。

10. **界面转场**  
    统一 `ScreenTransition` 工具：渐隐、中心展开、交叉淡入淡出；主页 ↔ 游戏 / 选关用柔和 crossfade，转场期间 `_transitioning` 防连点。见 `Scripts/ui/screen_transition.gd`、`Scripts/main.gd`。

## 5. 遇到的问题与解决方案

### 5.1 移动端触摸与 UI

| 问题 | 原因 | 解决 |
|------|------|------|
| 点牌无反应 | HUD 触摸区 `STOP` 挡住牌桌 | 对局区 `IGNORE`，`main.gd` 转发到 `Board` |
| 点一次选上又取消 | 触摸模拟鼠标双发 | `PointerInput` 同帧去重 |
| 选关滑动误触进关 | 按钮自己处理 `gui_input` | `ScrollListTap` + 行节点 `IGNORE` |
| 在按钮上滑不动列表 | 子节点 `mouse_filter=STOP` 抢手势 | 同上，手势交给 `ScrollContainer` |

### 5.2 布局与显示

| 问题 | 原因 | 解决 |
|------|------|------|
| 三层牌像三行平铺 | 层间距过大、`y_sort` 盖层 | 收紧 `GRID_Y_STEP`，用 `z_index` 控前后 |
| 超长屏黑边 | `stretch/aspect` 不合适 | 改为 `keep_width`，牌桌 `BoardLayoutScaler` 自适应 |
| 改 `ui_constants` 顶栏不动 | 代码 `_ready` 覆盖场景 | 删除运行时改布局，**只在场景里调** |
| 结算文案竖排 | `Label` 宽度被压成一字宽 + `autowrap` | `size_flags` 改为横向铺满 |

### 5.3 代码与引擎

| 问题 | 现象 | 解决 |
|------|------|------|
| `GridSlots` 找不到 | 脚本里写了 `//` 整除 | GDScript 改为 `/` |
| `button_press_scale` 报错 | 无 meta 时 `get_meta` | 先 `has_meta()` |
| 粒子出现在 (0,0) | 碰撞坐标在 `TileLayer` 本地空间，效果误挂 `Board` | 改挂 `TileLayer`，碎屑改用 `Polygon2D` |
| 碰撞闪大白块 | 冲击闪光 `Polygon2D` 过大过亮 | 去掉闪光层，缩小并调暗粒子 |
| 连击提示重复 | 消除点与屏幕各弹一次 | 只保留右侧 `ComboTip` 单次动画 |
| 主界面切换生硬 | 全屏 `SCALE_IN` + 先藏后显 | 主页改用 crossfade 交叉淡入淡出 |

### 5.4 资源管线（耗时最多）

| 阶段 | 问题 | 处理 |
|------|------|------|
| AI 生图 | 风格不一、字牌糊、小尺寸难认 | 作参考，不全信 |
| 自动切图 | 尺寸不一、命名混乱 | 放弃全自动，改 PS |
| PS 手修 | 34 张牌 + 三套 UI 工作量大 | 统一画布、居中、调色、按 `TEXTURE_MAP` 命名导入 |
| 引擎兜底 | 缺图或坏图 | `TileTextureAtlas` 回退程序化绘制，保证不崩 |

**教训**：手游 UI 不要指望「生一大张 → 自动切 34 张」一步到位；应先定规范再进引擎。

### 5.5 程序化关卡生成

每关进局时走 `RuntimeLevelBuilder.generate()` → `LayoutGenerator` / `GridSlots` 摆位 → `LevelChecker.verify_solvable()` → `TileAssigner` + `BoardSolver`。开发过程中踩过的坑：

| 问题 | 原因 | 解决 |
|------|------|------|
| `GridSlots` 类加载失败 | 脚本里误写 `//` 整除，整文件语法错误 | 改为 `/`；Godot 报「找不到 class_name」要先打开该类查语法（见 5.3） |
| 手写 JSON 难维护 20 关 | 每关自写 `cells`，L1 半格对齐极易错 | 固定 **5×7 槽位**：L0 最多 35、L1 半格偏移最多 24、L2 最多 15 |
| 上层牌随便摆坐标 | 麻将叠层要求 L1 坐在 L0 **相邻两牌中间** | `GridSlots._filter_layer1_on_l0` / `_filter_layer2_on_l1` 过滤合法槽位 |
| 某「层数 + 牌数」凑不出拓扑 | L1/L2 候选池不够或分布不合法 | `build_layout` 返回空 → `build_layout_attempt` 换层分布枚举 |
| 布局合法但牌型无解 | 纯随机配对绝大多数不可通关 | `TileAssigner`：**剥洋葱倒推**（`_assign_by_peeling`）优先，失败再随机洗牌 + DFS |
| 目标牌数死活不可解 | 固定 `tile_count` 与拓扑 / 牌池冲突 | `BUMP_DELTAS` 按 +0、+2、+4 交替**加牌重试**（最多 10 轮） |
| 高关卡生成慢或 DFS 卡住 | 40+ 格搜索空间大 | 运行时 `RUNTIME_MAX_ATTEMPTS=128`、`RUNTIME_MAX_NODES` 上限（编辑器校验可用 500 次） |
| 奇数格无法配对 | 消除规则要求成对 | `_try_tile_count` 发现奇数自动 **+1** |
| 底层行末剩 1～2 格难看 | 视觉像缺一块 | `_is_nice_l0_count`：L0 列数余数须为 0 或 ≥3 |
| 牌池小于对数 | 关卡只抽部分牌型 | `assign` 用 `pool[index % pool.size()]` 循环复用 |
| 偶发仍生成失败 | bump 与分布候选都耗尽 | `push_warning`；`board.gd` `push_error` 并 `_playing = false` |
| P4 仍绑 `layout_path` | 早期每关一份 `demo_xx.json` | 演进到 `LevelCatalog.build_level()` + 运行时生成，`layout_path` 留空；`levels.json` 只配 id / 名 / 难度 |
| 洗牌后布局变了 | 洗牌 = 重新走生成管线 | `regenerate_level` 同样调用 `RuntimeLevelBuilder`，动画由 `ShuffleEffect` 收拢 / 展开 |

**生成管线（简图）**：

```
level_id → 层数(1/2/3) + 目标牌数(8+(id-1)*2)
  → GridSlots 按层分布摆 cells
  → TileAssigner.assign_solvable（剥洋葱 → 随机+DFS）
  → 失败则 bump 牌数或换分布，直到 ok 或放弃
```

**教训**：「能摆盘」和「能通关」是两件事——布局与牌型分配必须分开，且生成层要有**重试 + 可解验证**；不要指望一次随机就过关。

## 6. 已知不足

- **美术**：牌面与参考 Vita 仍有差距，部分牌靠逐张 `scale` 对齐。
- **刘海屏**：已移除代码侧安全区适配，极端机型顶栏可能偏贴边（可在场景里加大 `margin_top`）。
- **热更**：启动会 `await HotUpdateManager`，无 CDN 时仅快速跳过，对作业无影响。

## 8. OpenSpec 规格汇总

项目用 **OpenSpec spec-driven** 记录各阶段需求；实现以 `openspec/specs/` 为真相来源，变更过程归档在 `openspec/changes/archive/`。总设计见 `openspec/design.md`。

### 8.1 分阶段归档（P0 → P5 + 体验补丁）

| 阶段 / 变更 | 归档目录 | 新增或更新的能力 spec |
|-------------|----------|------------------------|
| **P0** 单牌白模 | `2026-06-13-vita-mahjong-p0-tile-setup` | `tile-display` |
| **P1** JSON 布局 + 建盘 | `2026-06-13-vita-mahjong-p1-layout` | `layout-template`、`board-generation` |
| **P2** 点选消除玩法 | `2026-06-13-vita-mahjong-p2-gameplay` | `tile-gameplay`（可选牌、配对、提示、通关） |
| **P3** 随机洗牌 + 可解性 | `2026-06-13-vita-mahjong-p3-tile-assignment` | `tile-assignment` |
| **P4** 20 关与进度 | `2026-06-13-vita-mahjong-p4-levels` | `level-progression`；扩展 `board-generation` / `tile-gameplay` |
| **P5** 贴图与动效打磨 | `2026-06-14-vita-mahjong-p5-polish` | `tile-visuals` |
| **选关滑动** | `2026-06-14-vita-ui-scroll-list-tap` | 合并进 `level-progression`（ScrollListTap） |
| **手感二次打磨** | `2026-06-14-vita-gameplay-feel-polish` | `tile-gameplay`（连击）、`tile-visuals`（选中/碎裂）、**新建** `game-ui` |

演进路径：**白模单牌 → 固定 JSON 布局 → 玩法闭环 → 可解洗牌 → 多关进度 → 视听打磨 → 连击/碰撞/设置等大控件**。

### 8.2 现行 8 份能力 spec（`openspec/specs/`）

| Spec | 职责摘要 | 关键需求点 |
|------|----------|------------|
| **tile-display** | 牌型数据与白模展示 | 34 种 `TileRegistry`；`MahjongTile.setup()` 显示名 + 花色底色；`set_selected` 高亮 |
| **layout-template** | 布局 JSON 格式 | `{name, cells[{x,y,layer}]}`；`LayoutLoader` 校验非空与字段完整 |
| **board-generation** | 网格 → 世界坐标与建盘 | ×2 逻辑格、半格偏移、`z_index = layer*100+grid_y`；`BoardBuilder` cells 与 tile_ids 数量一致 |
| **tile-assignment** | 牌型分配与可解 | N/2 对 shuffle；`assign_solvable` + `BoardSolver` DFS；开局自动分配、R 键重洗 |
| **tile-gameplay** | 核心玩法规则 | `FreeTileChecker`；金黄选中 + 绿配对提示；3 秒连击窗口与加分；`level_cleared.max_combo` |
| **level-progression** | 关卡与进度 | `levels.json` / `LevelRegistry`；选关 UI、解锁存档；**ScrollListTap** 滑动与点击分离 |
| **tile-visuals** | 牌面与消除表现 | 程序化/贴图牌面；金选中框 + 绿提示；撞合碎裂、粒子挂 `TileLayer`；挡牌飘字、SFX |
| **game-ui** | HUD 与弹窗 UI | 右侧单次连击提示；结算最高连击 + 进度条；设置大按钮与橙色音量条 |

### 8.3 与代码的对应关系

- **数据层**：`TileType`、`TileRegistry`、`CellData`、`LevelData`、`LevelCatalog`
- **生成层**：`LayoutLoader`（仍支持 JSON）、`GridSlots`、`LayoutGenerator`、`RuntimeLevelBuilder`、`TileAssigner`、`BoardSolver`、`BoardBuilder`
- **表现层**：`MahjongTile`、`Board`、`GameHUD`、`ResultOverlay`、`ScreenTransition`

说明：P4 spec 仍描述 `layout_path` 驱动关卡；**当前实现**已改为 `RuntimeLevelBuilder` 程序化生成（见 §2.3、§5.5），`levels.json` 仅保留元数据。洗牌动画与界面转场尚未单独归档 spec，实现见 `shuffle_effect.gd`、`screen_transition.gd`。

## 9. 总结

本项目在 **24 小时量级** 内完成了 Vita Mahjong 核心三屏流程、20 关进度与移动端可玩版本。开发上采用 **AI 辅助编码 + 人工场景与美术修图 + OpenSpec 记录** 的组合；体验上重点解决了 **触摸链路、选关滑动误触、叠层布局、程序化关卡可解生成、连击反馈、选中辨识度、碰撞碎裂感、洗牌动画、界面转场、设置控件可点性** 等真实手游问题。

OpenSpec 从 P0 到 P5 共 8 份能力 spec（§8.2），手感与 UI 补丁已归档；生成层难点已记入 §5.5。
